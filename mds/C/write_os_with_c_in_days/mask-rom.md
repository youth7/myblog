在 RISC‑V 里，**Mask ROM（掩膜只读存储器，常被称作 BootROM）** 是集成在芯片内部、**出厂时用光刻掩膜把代码永久固化**的一小块 ROM，上电后 CPU 第一条指令就从这里开始执行，**不可改写、不可擦除、非易失**。

### 一、本质与特点

- **Mask = 掩膜**：芯片制造时用定制光刻版，直接把 0/1 做在硅片里，不是事后烧录。
- **ROM = 只读**：出厂后**永远不能改**，软件、工具都写不进去。
- **非易失**：断电不掉数据，一直存在。
- RISC‑V 里的核心作用：
  - 是整个系统**最早、最底层、最可信的代码**，属于硬件级 “信任根”。
  - 大小通常很小（几 KB 到几十 KB），只放最基础的初始化和加载逻辑。

### 二、里面一般放什么（RISC‑V 典型）

- 基础硬件初始化：关闭看门狗定时器（WatchDog Timer，WDT）、设置时钟频率、初始化串口 / USB 等。
- 启动流程第一阶段：
  - 从 Flash/SD/USB 读取并校验 **第一级 BootLoader（FSBL/SPL）**。
  - 校验签名（如果是安全启动），保证下一级代码没被篡改。
- 异常兜底：当 Flash 里的 BootLoader 损坏时，进入 **Mask ROM 模式**，通过 USB 下载新 BootLoader 修复 “变砖”。

### 三、和其他启动相关存储的区别

- **Mask ROM（BootROM）**：工厂固化，不可改；RISC‑V 上电 0 地址；最小、最底层。
- **OTP/EFUSE**：一次性可编程，出厂后可写一次；存密钥、配置、安全标志。
- **Flash（Nor/Nand）**：可多次擦写；放 U‑Boot、OpenSBI、内核、文件系统。

### 四、在 RISC‑V 启动链里的位置

```plaintext
CPU 上电 → Mask ROM（BootROM，不可改）
  → 校验/加载 FSBL（第一级 BootLoader）
  → 加载 OpenSBI/U‑Boot
  → 加载 Linux 内核
  → 根文件系统
```

Mask ROM 是**信任链的起点**，只要它没被攻破，后面每一步都可以被它验证。

### 五、为什么用 Mask ROM（而不是 Flash）

1. **绝对不可篡改**：硬件固化，软件完全无法修改，适合放安全根。
2. **上电必启动**：不管外部 Flash 有没有、坏没坏，Mask ROM 一定能跑，是 “救命” 代码。
3. **成本低、可靠性高**：大批量生产时便宜，抗干扰强。

### 六、常见叫法

- **Mask ROM**（正式）
- **BootROM**（因为主要放 Boot 代码）
- **ROM Code**、**MaskROM 模式**（修复变砖时用）

一句话总结：**RISC‑V 的 Mask ROM 就是芯片出厂时永久固化的一小段启动固件，上电第一个执行，不可改写，是系统启动和安全信任的硬件根。**
